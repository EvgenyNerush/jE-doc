Returns proposal density, which for a pair (rng_state, x), advances rng_state and x such that x = x + dx with dx found as described below. Note that if x is a tuple, the equation x = x + dx should be considered element-wise, and advance of rng_state* is doing for every element of the tuple x.

Parameters
rnga random number generator which takes and advances its state
aan amplitute to be used by method m
ma methot to produce dx from rng_state and a. To get dx evenly distributed in (-a, a) use appropriate *_cast_with_amplitude function from rng.hpp (i.e. pm_cast_with_amplitude() if pm_rng() is used). Note also that the proposal distribution should be symmetric in the metropolis algorithm that constrains the choose of m.
